# Yeoul

*[English](README.md)*

> *여울 — 개울이 돌 위로 얕고 빠르게 흐르는 구간 — 은 흩어진 생각이 부딪고 빨라져 정직하게 시험받는 곳이다.
> (영어권에는 이 지형을 뜻하는 "riffle"로 옮긴다.)*

**Yeoul은 심의 → 사전등록 → 자율 작업 루프를 자기기만과 성급한 종결에 저항하는 무결성 게이트와 함께
돌리는 파일 기반 하네스입니다.** 측정 규율 프리미티브(사전등록 + 변조 감지 원장) 위에 얹힌 *실천 계층*이에요 —
프리미티브는 *"이 주장이 정직한가?"*에 답하고, Yeoul은 *"규율 있는 아이디어 루프를 처음부터 끝까지 어떻게 돌리나?"*에 답합니다.

> ⚠️ **진행 중 — 아직 정식 릴리스 아님.** 이 문서는 포지셔닝 초안이지 출시 페이지가 아닙니다.

## 무엇인가 (그리고 아닌가)

- **맞다**: 다역할 심의 아크를 열고, 폭주 가드 아래 수렴을 판정하고, 정직하게 종결하고(빈칸 거부 + KILL 방어
  체크리스트), 기계 검증되는 개발 루프로 넘기는 — 작은 셸 스크립트 + 템플릿 + 방법론의 묶음.
- **아니다**: 새로운 멀티에이전트 프레임워크. 오케스트레이션은 일부러 평범하게 두었습니다 — 가치는 *게이트*에 있지
  토론에 있지 않아요. 연구에 따르면 토론 라운드를 늘려도 답 품질은 **오르지 않습니다**; Yeoul의 라운드/토큰/무진전
  가드는 토론이 스스로 교정해서가 아니라 *표류하기* 때문에 존재합니다.

## 게이트 (진짜 가치)

- **Gate-1 (선행연구)** — 기존 연구가 이미 다룬 아이디어는 초기에 기각하거나 각도를 튼다.
- **kill 조건 우선** — 반증조건 없는 명세는 검증 불가. 연산을 쓰기 *전에* 봉인한다.
- **2단계 종결** — 종결은 요약 초안을 만들고, 빈칸이 채워져야만 봉인된다.
- **KILL 방어 5항 체크** — "실패" 판정은 앵커 재현·독립각도 ≥2·구현결함 배제·도감 대조·kill 문언 일치가 전부
  기록되기 전엔 봉인되지 않는다.
- **검증 게이트 개발 루프** — 각 자동 개발 라운드는 TODO 한 항목을 진행하고, 그 항목의 기계 검증 명령이 exit 0일
  때만 체크할 수 있다. 검증 명령 없는 항목은 거부된다.

## 계층

| 계층 | 무엇 | 위치 |
|---|---|---|
| 프리미티브 | 사전등록 + 변조 감지 원장 | *(선택 의존성 — mirror-stack)* |
| 실천 | 심의 아크·게이트·개발 핸드오프 | **이 레포** |

mirror-stack 의존성은 선택입니다: 없으면 봉인이 no-op으로 완화되고 나머지는 그대로 돕니다.

## 상태 / 정직 노트

- 자율 개발 루프("Ralph" 방식)는 **가장 덜 검증됐고 가장 commodity한** 부분입니다. 그 안전성은 루프 자체가 아니라
  기계 검증 게이트에서 나와요. 검증된 기법으로 과신하지 마세요.
- Yeoul이 연구 *결과*를 개선한다는 주장은 하지 않습니다. 주장은 더 좁아요: 정직한 규율을 **기본값이자 건너뛰기
  어렵게** 만든다는 것 — 현재 자율 연구 시스템에 정확히 빠져 있는 바로 그것.
